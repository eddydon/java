Custom colors for events

To set a custom color for event(s) you can follow one of the following ways:

  1. To set colors in properties of the event object
  2. To apply additional css classes to the event(s)

Specifying color values in properties of the event object

The first way to set background and font colors for events is to add 2 additional properties to the data model (or just one of them):

  • textColor - specifies the font color of the event;
  • color - specifies the background color of the event.

Note, textColor and color are special properties. Each time when you populate planner with data, planner checks whether loaded events have these properties and if they do, applies the specified colors to the related events. If the properties aren't specified for a event, planner applies the predefined color to it.

Color values for the properties can be set using 3 different notations:

  • Hexadecimal Colors (e.g. ”#787878”);
  • Predefined/Cross-browser color names (e.g. “blur”)
  • RGB Colors (e.g. “rgb(80,0,0)”).
package demoapp;
 
import com.DHTMLX.Planner.DHXEvent;
 
public class ColoredEvent extends DHXEvent {
 
        public String textColor;
        public String getTextColor() {
                return textColor;
        }
        public void setTextColor(String textColor) {
                this.textColor = textColor;
        }
 
        public String color;
        public String getColor() {
                return color;
        }
        public void setColor(String color) {
                this.color = color;
        }
}

Attaching additional css classes to events

One more way to color events is to attach additional css classes to them.
Css classes are attached to events through the event_class template (see details of the planner template here)

By default, the template has definition as in: 

planner.templates.event_class = function(start, end, ev){
        return "";
}

The function takes 3 parameters:

  • start - the date when an event is scheduled to begin.
  • end - the date when an event is scheduled to be completed.
  • ev - the event object.

And returns a string which will be appended to the event class.

Let's assume you want to sort out your events and highlight the most important ones. For this, you decide to color events in different colors: important events - in red color, others - in blue.

So, you do the following:

  1. Add an additional property to the data model with the name, for example 'priority'. The property will store the priority of events : 'high' or 'normal'.



  2. Define css classes for the specified groups. Let's name these css classes as 'priority_event' and 'normal_event:

    <style>
        /*css classes for events in the Day and Week view*/
        .dhx_cal_event.priority_event div{
            background-color: #F20765 !important;
            color: #332C2F !important;
        }
        .dhx_cal_event.normal_event div{
            background-color: #6DABFC !important;
            color: #332C2F !important;
        }
 
        /*css classes for multi-day events in the Month view*/
        .dhx_cal_event_line.priority_event{
            background-color: #F20765 !important;
            color: #332C2F !important;
        }
        .dhx_cal_event_line.normal_event{
            background-color: #6DABFC !important;
            color: #332C2F !important;
        }
 
        /*css classes for events with the fixed time in the Month view*/
        .dhx_cal_event_clear.priority_event{
            color: #332C2F !important;
        }
        .dhx_cal_event_clear.normal_event{
            color: #332C2F !important;
        }
</style>
  3. And redefine the event_class template:

    planner.templates.event_class = function (start, end, event) {
           if (event.priority == 'high') return "priority_event";
           return "normal_event"; 
};
