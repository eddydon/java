Multiple planners on the page

 
This functionality included only in Scheduler PRO ( Enterprise license )

By default, when you render and initialize planner on the client, the library creates a DHXPlanner object with the name 'planner' in the global scope (i.e. it can be accessed as window.planner).
If you render multiple planners on the page, each next one will overwrite the global 'planner' object. In the upshot, one instance of DHXPlanner will exist on the page at a time.

To have several planners on the page and prevent their overwriting, you should specify different names for DHXPlanner instances. For this purpose, use the DHXPlanner.setName(String name) method: 

DHXPlanner planner1 = new DHXPlanner("./static/", DHXSkin.TERRACE);
planner1.config.setMultiDay(true);
 
DHXPlanner planner2 = new DHXPlanner("./static/", DHXSkin.TERRACE);
planner2.setName("planner2");
planner2.setInitialDate(2012, 8, 20);

The name of a DHXPlanner object can be gotten by using the DHXPlanner.getName() method: 

DHXPlanner planner1 = new DHXPlanner("./static/", DHXSkin.TERRACE);
String name = planner1.getName(); // -> 'planner'
