Timeline view

Initialization

DHXTimelineView view = new DHXTimelineView("type", "event_type", "Type");//initializes the view
view.setRenderMode(DHXTimelineView.RenderModes.BAR);
view.setXScaleUnit(DHXTimelineView.XScaleUnits.MINUTE);
view.setServerList("event_type");//defines Y-Axis items. 
...
planner.views.add(view);//adds the view to the planner

where

  • type - the name of the view.
    The constructor requires the 'name' cause the view can exist in the planner in several instances.
  • event_type - the name of a data property that will be used as the id of units.
    Optional. If the parameter isn't specified, it takes value of the first parameter.
  • Type - the text label of the view tab.
    Optional. If the parameter isn't specified, it takes value of the first parameter.

Main methods of the DHXTimelineView class

  • setDy(Integer height) / getDy() - sets/gets the minimal height of cells (if the SectionAutoheight parameter has value 'false', the height of cells will be equal to Dy. Otherwise, the height will be increased to fill all free space).
  • setDx(Integer width) / getDx() - sets/gets the width of the first column (that contains names of units).
  • setEventDy(Integer height) / getEventDy() - sets/gets the height of the event box.
  • setEventMinDy(Integer height) / getEventMinDy() - sets/gets the minimum height of the event box.
  • setFilter(DHXFilter filter) / getFilter() - defines filter for the view/gets the filter set for the view.
  • setFullEventDy(Boolean occupyAllSpace) / getEventMinDy() - specifies whether the event must occupy the entire cell (ignores the value set by setEventDy(Integer height))
  • setFitEvents(Boolean isHeightFlexible) / getFitEvents() - defines whether the height of cells should be flexible or fixed (setDy(Integer height) method). The default value - true.
  • setFolderDy(Integer height) / getFolderDy() - sets/gets the height of folders in pixels (non-applicable to cells with folder names).
  • setFolderEventsAvailable(Boolean createEventForGroup) / getFolderEventsAvailable() - call the method with the true value if you want to have the possibility to specify events not only for an individual holder but also for the whole folder (any level). Used for the 'Tree' mode.
  • setLabel(String label) / getLabel() - sets/gets the text label. The default value - 'Timeline'.
  • setName(String name) / getName() - sets/gets the name of the view. Most applicable to the Units and Timeline views cause they can be presented in the planner in several instances. For other views 'name' is taken by default.
  • setYProperty(String property) / getYProperty() - sets/gets the name of a property by which events are divided to units.
  • setRenderMode(RenderModes mode) / getRenderMode() - sets/gets the mode of the view.
  • setResizeEvents(Boolean fitSection) / getResizeEvents() - specifies whether the height of events must be decreased to fit the section height. By default - true.
  • setSectionAutoheight(Boolean cellAutoSection) / getSectionAutoheight() - allows you to set automatic height adjustment for cells.
  • setSkipIncorrect - (boolean) skips (doesn't display) events which belong to none of the defined units. If the parameter is set to false, such events are presented in the first unit. The default value - false.
  • setTabPosition(Integer offset) / getTabPosition() - sets/gets the right offset of the view tab in the tabbar. By default, tabs go right-to-left in the order of addition to the planner.
  • getViewType() - gets the type of the view.
  • setXDate(String format) / getXDate() - sets/gets the date format of the X-Axis. See available variations of the format string here.
  • setXLength(Integer steps) / getXLength - sets/gets a number of 'XStep's needed to fill the whole day.
  • setXSize(Integer minInterval) / getXSize() - sets/gets the minimum scale interval calculated in 'XUnit's.
  • setXStart(Integer minValue) / getXStart() - sets/gets the minimal value of the scale.
  • setXStep(Integer steps) / getXStep() - sets/gets the number of 'XStep's needed to display on the screen.
  • setXScaleUnit(XScaleUnits unit) / getXScaleUnit() - (minute, hour, day, week, month, year ) the scale unit.
  • setYProperty(String name) / getYProperty() - the event property that will be used as a template for Y-Axis items.
DHXTimelineView view = new DHXTimelineView("type", "event_type", "Type");
view.setRenderMode(DHXTimelineView.RenderModes.BAR);
view.setXScaleUnit(DHXTimelineView.XScaleUnits.HOUR);
view.setXStep(30);
view.setXSize(24);// (8PM - 8AM)/30min
view.setXStart(16);// 8AM/30min
view.setXLength(48);// 24/30min
view.setServerList("event_type");
planner.views.add(view);

Second X-Axis

The second X_Axis is placed in the top of the default one and serves to group time intervals of the original scale.

You can add it with method AddSecondScale(XScaleUnits x_unit, String x_date).

The method takes 2 parameters:

  1. The scale unit. Possible values are minute, hour, day, week, month, year.
  2. The date format. See available variations of the format string here. 
DHXTimelineView view = new DHXTimelineView("type", "event_type", "Type");
view.addSecondScale(DHXTimelineView.XScaleUnits.DAY,"%F %d");

Modes

The view can be used in 3 modes:

  • Bar
  • Cell (default value)
  • Tree

The desired mode is set by calling method view.setRenderMode(RenderModes mode). 

DHXTimelineView view = new DHXTimelineView("type", "event_type", "Type");
view.setRenderMode(DHXTimelineView.RenderModes.BAR);

Data for Y-Axis

'Bar' and 'Cell' modes

There are 2 ways, you can set values for Y-Axis in the 'Bar' and 'Cell' modes:

  • With method DHXTimelineView.addOption(DHXTimelineUnit unit):

    DHXTimelineView view = new DHXTimelineView("type", "event_type", "Type");
...
view.addOption( new DHXTimelineUnit("1", "Department 1"));
  • With method DHXTimelineView.setServerList(String list):

    DHXTimelineView view = new DHXTimelineView("type", "event_type",  "Type");
...
view.setServerList("event_type");

    where “event_type” is ArrayList<DHXCollection> specified in the java file that answers for loading data:

    //CustomEventsManager.java
public class CustomUnitsEventsManager extends DHXEventsManager {
        ...
        @Override
	public HashMap<String, Iterable<DHXCollection>> getCollections() {
		ArrayList<DHXCollection> event_type = new ArrayList<DHXCollection>();
		event_type.add(new DHXCollection("1", "Simple"));
		event_type.add(new DHXCollection("2", "Complex"));
		event_type.add(new DHXCollection("3", "Other"));
 
		HashMap<String,Iterable<DHXCollection>> c = 
                          new HashMap<String,Iterable<DHXCollection>>();
		c.put("event_type", event_type);
		return c;
	}
}

Note, to be correctly processed data items must have 2 mandatory properties:

  • key - the id of an item;
  • label - the text label of an item.

'Tree' mode

The 'Tree' mode allows you to present hierarchical structures and show data in expandable groups.

There are 2 ways, you can set values for Y-Axis in the 'Tree' modes.

  1. With method DHXTimelineView.addOption(DHXTimelineUnit unit)
  2. With method DHXTimelineView.setServerList(String list)

Using the DHXTimelineView.addOption(DHXTimelineUnit unit)

While using the DHXTimelineView.addOption(DHXTimelineUnit unit) you have 2 variants to choose from:

  • To set each scale item individually.

    var timeline = new TimelineView("timeline", "room_id");
timeline.RenderMode = TimelineView.RenderModes.Tree;
 
var section = timeline.AddOption( new TimelineUnit("1", "Department 1", true));//adds folder to the axis          
section.AddOption( new TimelineUnit("2", "Room 1") );// defines the items of the folder
section.AddOption( new TimelineUnit("3", "Room 2") );
 
var section2 = timeline.AddOption( new TimelineUnit("4", "Department 2", true));
section2.AddOption( new TimelineUnit("5", "Room 1") );
section2.AddOption( new TimelineUnit("6", "Room 2") );

    where the TimelineUnit constructor takes 3 parameters:

    • The id of a folder;
    • The text label of a folder;
    • Defines whether a folder will be opened initially (optional).

  • To set all items at once. 

    var context = new DHXPlannerDataContext();
var timeline = new TimelineView("timeline", "room_id");
timeline.FolderEventsAvailable = false;
timeline.RenderMode = TimelineView.RenderModes.Tree;
foreach (var department in context.Departments)
{
        TimelineUnit section = new TimelineUnit(department.Id, department.Title);// defines the header of the folder
        timeline.AddOption(section);
        section.AddOptions( department.Rooms );//'Rooms' in the name of model data table
}
planner.Views.Add(timeline);

    Note, data items have 2 mandatory properties (the same as for the Bar and Cell modes): key and label.

Using the DHXTimelineView.setServerList(String list) method

Here you should remember an important thing:

  • Sub-items are specified through the children property. As, by default, the DHXCollection class doesn't allow to retrieve such a property, you should extend the DHXCollection class. Your custom class may look like this:

    public class Table extends DHXCollection {
 
    public ArrayList<DHXCollection> children = null;
    public Iterable<DHXCollection> getChildren() {
        return children;
    }
    public void setChildren(ArrayList<DHXCollection> children) {
        this.children = children;
    }
    public void addChild(DHXCollection child) {
      if (children == null)
          children = new ArrayList<DHXCollection>();
          children.add(child);
    }
 
}

    Actually, by redefining the DHXCollection class you can retrieve any extra properties from the server, e.g. some user data. 



DHXTimelineView view = new DHXTimelineView("type", "table_id",  "Type");
...
view.setServerList("tables_tree");

where “tables_tree” is ArrayList<DHXCollection> specified in the java file that answers for loading data:

//CustomEventsManager.java
public class CustomUnitsEventsManager extends DHXEventsManager {
        ...
        @Override
	public HashMap<String, Iterable<DHXCollection>> getCollections() {
		ArrayList<DHXCollection> tables_tree = new ArrayList<DHXCollection>();
 
                Table c1 = new Table("1-2", "1-2 people");
                Table c2 = new Table("3-6", "3-6 people");
                c1.addChild(new Table("table1", "Table 1"));
                c1.addChild(new Table("table2", "Table 2"));
                c2.addChild(new Table("table5", "Table 5"));
                c2.addChild(new Table("table6", "Table 6"));
 
                tables_tree.add(c1);
                tables_tree.add(c2);
 
		HashMap<String,Iterable<DHXCollection>> c = 
                          new HashMap<String,Iterable<DHXCollection>>();
		c.put("tables_tree", tables_tree);
		return c;
	}
}

Generated JSON will look like this: 

{
   data: [ ],
   collections: {
      tables_tree: [
        {
            value: "1-2",
            label: "1-2 people (2 people, smoking)",
            children: [
                { value: "table1", label: "Table 1 (2 people, smoking)"},
                { value: "table2", label: "Table 2 (2 people, smoking)"}
        ]},
        {
             value: "3-6",
             label: "3-6 people (5 people, smoking)",
             children: [
                 { value: "table5", label: "Table 5 (4 people, smoking)" },
                 { value: "table6", label: "Table 6 (4 people, smoking)" }
             ]
        }
   ]}
}
