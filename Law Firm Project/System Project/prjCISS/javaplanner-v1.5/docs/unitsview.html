Units view

Initialization

DHXPlanner planner = new DHXPlanner("./static/", DHXSkin.TERRACE);
...
DHXUnitsView view = new DHXUnitsView("type", "event_type", "Type");
view.setServerListLink("event_type");
planner.views.add(view);

where

  • type - the name of the view.
    The constructor requires the 'name' cause the view can exist in the planner in several instances.
  • event_type - the name of a data property that will be used as the id of units.
    Optional. If the parameter isn't specified, it takes value of the first parameter.
  • Type - the text label of the view tab.
    Optional. If the parameter isn't specified, it takes value of the first parameter.

Main methods

  • DHXUnitsView.addOption(DHXUnit unit) - sets values for Y-Axis.
  • DHXUnitsView.setFilter(DHXFilter filter) / getFilter() - defines filter for the view/gets the filter set for the view.
  • DHXUnitsView.setLabel(String label) / getLabel() - sets/gets the text label. The default value - 'Units'.
  • DHXUnitsView.setName(String name) / getName() - sets/gets the name of the view. Most applicable to the Units and Timeline views cause they can be presented in the planner in several instances. For other views 'name' is taken by default.
  • DHXUnitsView.setProperty(String property) / getProperty() - sets/gets the name of a property by which events are divided to units.
  • DHXUnitsView.setServerListLink(String list) / getServerListLink() - specifies/gets the ArrayList<DHXCollection> object that is used to set values for Y-Axis.
  • DHXUnitsView.setSize(Integer units) / getSize() - sets/gets the number of units to be displaying on the screen at a time.
  • DHXUnitsView.setSkipIncorrect(Boolean skipIncorrect) / getSkipIncorrect() - skips (doesn't display) events which belong to none of the defined units. If the method is called with the false value, such events are presented in the first unit. The default value - false.
  • DHXUnitsView.setStep(Integer step) / getStep() - sets/gets the scroll step.
  • DHXUnitsView.setTabClass(String className) / getTabClass() - specifies/gets the name of a css class that will be applied to the tab.
  • DHXUnitsView.setTabPosition(Integer offset) / getTabPosition() - sets/gets the right offset of the view tab in the tabbar. By default, tabs go right-to-left in the order of addition to the planner.
  • DHXUnitsView.setTabStyle(String style) / getTabStyle() - specifies/gets the style that will be applied to the tab.
  • DHXUnitsView.setTabWidth(Integer width) / getTabWidth() - specifies/gets the width of the tab.
  • DHXUnitsView.getViewType() - gets the type of the view.

Data for X-Axis

There are 2 ways, you can set values for Y-Axis:

  • With method DHXUnitsView.addOption(DHXUnit unit):

    DHXUnitsView view = new DHXUnitsView("type", "event_type", "Type");
...
view.addOption( new DHXUnit("1", "Department 1"));
  • With method DHXUnitsView.setServerListLink(String list)

    DHXUnitsView view = new DHXUnitsView("type", "event_type", "Type");
...
view.setServerListLink("event_type");

    where “event_type” is ArrayList<DHXCollection> specified in the java file that answers for loading data:

    //CustomEventsManager.java
public class CustomUnitsEventsManager extends DHXEventsManager {
        ...
        @Override
	public HashMap<String, Iterable<DHXCollection>> getCollections() {
		ArrayList<DHXCollection> event_type = new ArrayList<DHXCollection>();
		event_type.add(new DHXCollection("1", "Simple"));
		event_type.add(new DHXCollection("2", "Complex"));
		event_type.add(new DHXCollection("3", "Other"));
 
		HashMap<String,Iterable<DHXCollection>> c = 
                        new HashMap<String,Iterable<DHXCollection>>();
		c.put("event_type", event_type);
		return c;
	}
}

Note, to be correctly processed data items must have 2 mandatory properties:

  • key - the id of an item;
  • label - the text label of an item.
