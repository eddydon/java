Data collections

In this chapter we will consider data collections in the context of static data loading (in the static mode, data is loaded once when the planner starts to run and doesn't require any additional requests to the server).

Let's consider 3 situations you may need to use a data collection:

  • To populate the planner with data;
  • To specify select options of a dropdown list;
  • To specify axis items in the Unit and Timeline views.

Events data collections

You can use data collections to load data into the planner. Data is loaded through the DHXPlanner.parse(Iterable <DHXEv> events_list) method: 

//CustomEventsManager.java file
public class CustomEventsManager extends DHXEventsManager {
    ...
    @Override
    public Iterable getEvents() {
          ArrayList events = new ArrayList();
 
          events.add(new DHXEvent(1, "05/13/2013 06:00", "05/13/2013 10:00", "Demo event #1"));
          events.add(new DHXEvent(2, "05/14/2013 06:00", "05/14/2013 10:00", "Demo event #2"));
          events.add(new DHXEvent(3, "05/15/2013 06:00", "05/15/2013 10:00", "Demo event #3"));
 
          return events;
    }
}
//Sampleplanner.jsp
DHXPlanner planner = new DHXPlanner("./static/", DHXSkin.TERRACE);
...
CustomEventsManager evs = new CustomEventsManager(request);
planner.parse(events.getEvents());

Note, to be correctly processed data items for the controls should have 4 properties:

  • id - (Integer, String etc.) the event id.
  • start_date - (Date or String) the date when an event is scheduled to begin. The default format - ”%m/%d/%Y %H:%i”.
  • end_date - (Date or String) the date when an event is scheduled to be completed. The default format - ”%m/%d/%Y %H:%i”.
  • text - (String) the event text.

Read chapter 'Format string' to find out what characters you can use while setting the format for dates.

Data collection in dropdown lists

Another use case of data collections is to specify select options for various dropdown lists, e.g. 'select' control. 

DHXLightboxSelect select = new DHXLightboxSelect("textColor", "Priority");
select.addOption("gray", "Low");
select.addOption("blue", "Medium");
select.addOption("red", "Low");

Note, the DHXLightboxSelect.addOption(String key, String value) takes 2 parameters:

  • key - the item id;
  • label - the item label.

Data collections for the Timeline view

Items for Y-Axis in the Timeline view can be specified by a data collection. 

DHXTimelineView view = new DHXTimelineView("type", "Type", "event_type");
...
view.addOption( new DHXTimelineUnit("1", "Department 1"));
view.addOption( new DHXTimelineUnit("2", "Department 2"));

Note, a DHXTimelineUnit objects take 2 parameters:

  • key - the item id;
  • label - the item label.

Data collections for the Units view

Items for X-Axis in the Units view can be specified by a data collection. 

DHXUnitsView view = new DHXUnitsView("type", "event_type", "Type");
...
view.addOption( new DHXUnit("1", "Room 1"));
view.addOption( new DHXUnit("1", "Room 2"));

Note, DHXUnit objects take 2 parameters:

  • key - the item id;
  • label - the item label.
