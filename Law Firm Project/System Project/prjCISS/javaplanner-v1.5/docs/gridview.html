Grid view

Initialization

DHXPlanner planner = new DHXPlanner("./static/", DHXSkin.TERRACE);
...
DHXGridView view = new DHXGridView();//initializes the view
view.addOption(new DHXGridViewColumn("text", "Text"));// adds a column to the grid
view.addOption(new DHXGridViewColumn("start_date", "Start date", 150));
view.addOption(new DHXGridViewColumn("end_date", "End date", 150));
planner.views.add(view);

Main methods

DHXGridView class

GridView

  • DHXGridView.getViewType() - gets the type of the view.
  • DHXGridView.addOption(DHXUnit unit) - specifies a column in the grid.
  • DHXGridView.addOptions(Iterable <DHXGridViewColumn> columns) - specifies columns of the grid.
  • DHXGridView.setColumns(ArrayList <DHXGridViewColumn> columns) - specifies columns of the grid. See details below.
  • DHXGridView.setFilter(DHXFilter filter) / getFilter() - defines client-side filtering for the view/gets the filter set for the view.
  • DHXGridView.setFrom(Calendar startDate) / getFrom() - sets/gets the left border of the allowable date range. Used to limit the displayable date range.
  • DHXGridView.setLabel(String label) / getLabel() - sets/gets the text label. By default - 'Grid'.
  • DHXGridView.setName(String name) / getName() - sets/gets the name of the view. By default - 'grid'.
  • DHXGridView.setPaging(Boolean enableNavigation) / getPaging() - enables/disables navigation in the grid with buttons: .
  • DHXGridView.setRowHeight(Integer height) / getRowHeight() - sets/gets the height of rows in the grid.
  • DHXGridView.setSelect(String label) / getSelect() - enables/disables selection in the grid (by default, selection is enabled).
  • DHXGridView.setTabClass(String className) / getTabClass() - specifies/gets the name of a css class that will be applied to the tab.
  • DHXGridView.setTabPosition(Integer offset) / getTabPosition() - sets/gets the right offset of the view tab in the tabbar. By default, tabs go right-to-left in the order of addition to the planner.
  • DHXGridView.setTabStyle(String style) / getTabStyle() - specifies/gets the style that will be applied to the tab.
  • DHXGridView.setTabWidth(Integer width) / getTabWidth() - specifies/gets the width of the tab.
  • DHXGridView.setTo(Calendar endDate) / getTo() - sets/gets the right border of the allowable date range. Used to limit the displayable date range.

DHXGridViewColumn class

  • DHXGridViewColumn.setAlign(Aligns align) / getAlign() - sets/gets the horizontal text alignment.
  • DHXGridViewColumn.setCustomSort(String function) - sets a custom sorting function for the column.
  • DHXGridViewColumn.setId(String id) / getId() - sets/gets the column id.
  • DHXGridViewColumn.setLabel(String label) / getLabel() - sets/gets the column header label.
  • DHXGridViewColumn.setSorting(Sort type) / getSort() - enables sorting for the column (triggered by a single click on the header) and assigns one of predefined sorting types.
  • DHXGridViewColumn.setTemplate(String template) / getTemplate() - sets/gets the data template for the column.
  • DHXGridViewColumn.setVAlign(VAligns align) / getVAlign() - sets/gets the horizontal text alignment.
  • DHXGridViewColumn.setWidth(Integer width) / getWidth() - sets/gets the column width. You can use '*' (asterisk) as the width value to force a specific column to fill up the remaining width. If you use '*' for several columns they will share out the remaining width equally.

Limiting date range

In the view you are allowed to limit active dates so that the users won't be able to get the dates beyond the range.

If, for example, you want to to limit active dates and set the allowable range from 1st January, 2012 to 1st January, 2013 you should provide the following code: 

DHXGridView view = new DHXGridView();
view.setFrom(2012, 0, 1);
view.setTo(2013, 0, 1);

Activating navigation

To activate navigation with buttons in the grid you should call the DHXGridView.setPaging(Boolean enableNavigation) method. 

DHXGridView view = new DHXGridView();
view.setPaging(true);

Sorting

When you click on the header, the grid starts to display a special control indicating which column the table is currently sorted by and the direction of this sorting (ascending or descending). Each next click on the same header will reverse the sorting direction.

Columns can have different type of content (numbers, strings, dates) and each type of content requires its specific way of sorting.

For this purpose, the library provides the DHXGridViewColumn.Sort class that sets 4 types of sorting:

  • Number;
  • Date;
  • String;
  • Custom.

To enable sorting and assign the appropriate sorting type to a column, you should call the DHXGridViewColumn.setSorting(Sort type) property. 

DHXGridViewColumn column1 = new DHXGridViewColumn("text", "Text");
column1.setSorting(DHXGridViewColumn.Sort.STRING)

Custom sorting functions

If you want to apply custom sorting behavior, you can define the related logic in a function and set this function as a parameter of the DHXGridViewColumn.setCustomSort(String function) function.

This function will be called for each pair of adjacent values and return 1,-1 or 0:

  • 1 - an object with the first value in pair must go before the second one;
  • -1 - the second object goes before the first one;
  • 0 - the objects are equal.

The function can be defined in a general way: 

DHXGridViewColumn column1 = new DHXGridViewColumn("start_date", "Start date");
column1.setSorting(DHXGridViewColumn.Sort.CUSTOM)	
...
column1.setCustomSort("customFunc");// 'customFunc' is the name of a custom JavaScript function that will be used for sorting

Templates

By default, columns display data of a property that is specified as Id of a column.
To customize data content you can use templates. In such a case, the column will be populated with data returned by the template function.
Data templates are set in the related column by calling method DHXGridViewColumn.setTemplate(String template). 

DHXGridViewColumn column1 = new DHXGridViewColumn("text", "Text");
column1.setTemplate("<% if(ev.start_date.valueOf() < new Date().valueOf()) { %>
               Expired
               <% } else { %>
               {start_date:date(%d-%m-%Y %H:%i)} 
               <% } %>");

The specified template generates a function that takes 3 parameters:

  • start_date - the start date of an event
  • end_date - the end date of an event
  • ev - the event object

Read more about templates in the related chapter of the documentation - DHXPlanner.templates.
