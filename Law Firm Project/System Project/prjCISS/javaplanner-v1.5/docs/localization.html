Calendar Language Localization

The library allows you to specify a Java calendar locale for a certain region by setting the required language for the user interface.

Locale activation

To set the necessary locale, you should call the DHXPlanner.localizations.set(DHXLocalization locale) method: 

DHXPlanner planner = new DHXPlanner("./static/", DHXSkin.TERRACE);
...			
planner.localizations.set(DHXLocalization.Spanish);
  • By default, locales reside in the directory path_to_planner_sources/locale.
    The path to the Java calendar sources are specified as the first parameter of the calendar constructor, e.g. in case of the constructor specified in the code snippet above, the path will be - ./static/locale/some_locale.js.
  • As you call the DHXPlanner.localizations.set(DHXLocalization locale) method, 2 new script files are included to your page automatically (one for classic events and the other - for recurring):

    <script src="./static/locale/locale_[languageName].js" type="text/javascript"></script>
<script src="./static/locale/locale_recurring_[languageName].js" type="text/javascript"></script>
  • Method set() can be also called as DHXPlanner.localizations.set(String locale) where locale is a short name of the locale (generally, it's an official two-letter code of the country), e.g. 'en' or 'ru'.

Recurring events

Note, if you aren't going to use recurring events in the Java calendar, you can pass the second parameter false to the DHXPlanner.localizations.set(DHXLocalization locale) method. In this case, just the locale_[languageName].js script file will be included to the page. 

DHXPlanner planner = new DHXPlanner("./static/", DHXSkin.TERRACE);
...			
planner.localizations.set(DHXLocalization.Spanish, false);

Custom locale

If the DHXLocalization class doesn't contain the required locale, you can create a new one.
Note, a locale file must be written in JavaScript. Detail information about creating custom locales you can read in the related chapter of the dhtmlxScheduler documentation - Localization. Creating custom locale.

To apply a custom locale to the calendar control, you should specify its short name in the DHXPlanner.localizations.set(String locale) method: 

DHXPlanner planner = new DHXPlanner("./static/", DHXSkin.TERRACE);
...			
planner.localizations.set("languageName");

Note, if you aren't going to use recurring events in the Java calendar, you can pass false as the second parameter to the DHXPlanner.localizations.set(DHXLocalization locale) method.

Let's assume, you have created a new locale for the Lettish language. To apply it to the calendar you should call: 

planner.localizations.set("lv");

Automatically, files locale_lv.js and locale_recurring_lv.js will be included to the page.


The DHXPlanner.localization class provides an important method:

  • setDirectory(String path) - sets the path to the locale files relative to the codebase directory.
