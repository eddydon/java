Read-only mode

The library provides support for the read-only mode in the planner.

You can set the read-only mode for:

  1. Whole planner;
  2. Lightbox;
  3. Certain events.

Read-only planner

To set the mode for the whole planner you should call the DHXPlanner.onfig.setReadonly(Boolean isReadonly) method with the true value. 

DHXPlanner planner = new DHXPlanner("./static/", DHXSkin.TERRACE);
...
planner.config.setReadonly(true);

Note, when you make the whole planner non-editable, users will have access only to the 'read' operation and can't open the lightbox.

Read-only lightbox

To allow users to open the lightbox but forbid any editing inside it, you need to call the DHXPlanner.config.setReadonlyForm(Boolean isReadonly) method with the true value: 

DHXPlanner planner = new DHXPlanner("./static/", DHXSkin.TERRACE);
...
planner.config.setReadonlyForm(true);

Read-only events

To prevent editing of specific events only, you should:

  • add to the data model a boolean property with the name - 'readonly':

    package demoapp;
 
import com.DHTMLX.Planner.DHXEvent;
 
public class ReadOnlyEvent extends DHXEvent {
    public Boolean readonly = false;
    public Boolean getReadonly() {
        return readonly;
    }
    public void setReadonly(Boolean readonly) {
        this.readonly = readonly;
    }
}
  • enable the Readonly extension:

    planner.extensions.add(DHXExtension.READONLY);

Now, each time the user edits some event, the planner checks whether this event has the readonly property set to true, and if it has - forbids editing.
